Hello World!

Available API Endpoints:

  • GET: /api/get-data
    • Returns currently available data.
    • If no data has been ever received from a device, example data is instead returned.
  • POST: /api/update-data
    • Send new volume info to the server.
    • Required Headers: Content-Type: application/json (if JSON) or Content-Type: text/plain (if semi-colon separated string).
    • Expects either a string or a JSON object.
      • If sending a string, the server expects a string with the format: {"volume": VOLUME, "id": ID}. (Order does not matter, quoted portions must match exactly. VOLUME must be a number.)
  • POST: /api/setup-device
    • Configure a device location and/or name.
    • Required Headers: Content-Type: application/json
    • Expects an object of the format:
      { "id": ID, "username": USERNAME, "lat": LATITUDE, "lon": LONGITUDE, "alt": ALTITUDE, "accuracy": ACCURACY, "name": DEVICE_NAME }
    • Only id and username are required. id must be the case-sensitive ID of a real known device (example devices wont work currently). USERNAME must be one of the following: "campbell", "clayton", "ryan", "nate", "austin" all lower-case.
    • All other values are optional, and can be excluded if you do not wish for the value to change.
    • Name, Username, and Id are Strings. lat, lon, alt, accuracy, are Numbers (or can be coerced to numbers).
  • DELETE: /api/delete-device
    • Delete all data about a given device.
    • Required headers: Content-Type: application/json
    • Expects an object of the format:
      { "id": ID, "username": USERNAME, }
    • Only id and username are required. id must be the case-sensitive ID of a real known device (example devices wont work currently). USERNAME must be one of the following: "campbell", "clayton", "ryan", "nate", "austin" all lower-case.
